# UI戦略 - 競合分析に基づく設計方針

## 競合UIの分析結果

### SessionLab（最大の間接競合）

**強み:**
- 1,616の手法をカード型グリッドで表示。タグ・フィルター・人気順ソート・著者情報付き
- タイムラインがドラッグ＆ドロップで並べ替え・差し替え可能
- テンプレートギャラリーにカバー画像付きカードで「完成形が見える」安心感
- 社会的証明（Airbus, Danone等のロゴ、テスティモニアル）を随所に配置
- カラーパレット: シアン・ブルー・パープルのグラデーション、白背景にカラフルなブロブイラスト
- タイポグラフィ: 42px見出し → 20px本文 → 13px補足の明確な3段階ヒエラルキー
- CTA戦略: 「Sign up for free」を複数箇所に配置、ブラウズ系の低コミットCTAも併設

**弱み（我々の勝ち筋）:**
- ゼロから手動設計が前提。自動生成機能がない
- 身体的・空間的アクティビティの設計に未対応
- 計画ツール止まり。当日の実行支援機能がない
- 英語中心。日本市場への最適化なし

### Howspace

**強み:**
- AI機能を「AI Sense-making」として独立セクションで紹介。核心機能として位置づけ
- ファーストビューにWalmart, Gucci, UN Womenのロゴで即座に信頼感
- デュアルCTA: 「Book a demo」（高コミット）+「Explore the product」（低コミット）
- カラーパレット: ディープパープル + ソフトピンク
- 24px基準のグリッドシステムで統一感

**弱み:**
- 大規模組織向け。個人ファシリテーターには過剰
- 高価格帯。気軽に試せない
- 体験型ワークには不向き

## 現状アプリの致命的なUI課題

| 課題 | 影響 | 優先度 |
|------|------|--------|
| ランディングが即フォーム | 初見で「何ができるのか」伝わらず離脱 | 最高 |
| 生成結果が編集不可 | 「たたき台」止まり。実務で使えない | 最高 |
| 保存もエクスポートもない | 一度閉じたら消える。リピート動機なし | 最高 |
| モバイルのタッチ操作が厳しい | スライダー6px、ボタン14px。現場利用に耐えない | 高 |
| 社会的証明ゼロ | 信頼感がなく、企業導入のハードルが高い | 高 |
| カラー9色以上混在 | 視覚的にノイズが多く、洗練度が低い | 中 |
| スペーシング不均一（14/16/18/20px混在） | デザインの一貫性が欠ける | 中 |
| ホバー効果がモバイル非対応 | タッチデバイスでUIが静的に見える | 中 |
| レベル選択に説明がない | 初級/中級/上級の違いがわからない | 中 |

## 勝つためのUI設計方針

### 1. ファーストビュー改革

**現状**: いきなり入力フォーム → **改善**: ヒーロー → デモ/生成例 → フォーム

```
[ヒーローセクション]
「演劇のプロが設計した400+のエクササイズから
 あなただけのワークショップを30秒で」

[生成例プレビュー] ← 実際の出力をアニメーションで見せる

[CTAボタン] 「無料で作ってみる」

[社会的証明] 導入実績・利用者の声（集まり次第追加）
```

### 2. 入力フォーム改善

- **ステッパーUI**: 4ステップを1画面ずつ表示。上部に進捗バー
- **各ステップに説明追加**: 「初級＝演劇未経験者向け」「上級＝即興に慣れた人向け」
- **リアルタイムフィードバック**: 選択のたびに「この条件で23のエクササイズが候補です」と表示
- **タッチ最適化**: スライダー高さ12px、つまみ44px、ボタン最低48x48px

### 3. 生成結果画面（最重要の改善箇所）

```
┌─────────────────────────────────────┐
│  タイムライン（横スクロール）            │
│  [アイスブレイク] → [メイン1] → [メイン2] → [クロージング]  │
│                                       │
│  ↕ ドラッグで並べ替え                   │
│  🔄 タップで別のエクササイズに差し替え    │
├─────────────────────────────────────┤
│  選択中のエクササイズ詳細               │
│  ┌──────────────────────────────┐    │
│  │ やり方  │  ねらい  │ 空間配置図 │    │ ← SessionLabにない独自要素
│  └──────────────────────────────┘    │
│  💬 トランジションガイド               │ ← 「次にこう声かけしてください」
│  ⏱ 所要時間: 20分                     │
├─────────────────────────────────────┤
│ [📄 PDF出力] [🔗 共有リンク] [▶ 進行モード] │
└─────────────────────────────────────┘
```

**ポイント:**
- エクササイズの入れ替え・並べ替えが可能（SessionLabのドラッグ＆ドロップに対抗）
- 空間配置図とトランジションガイドは演劇特化の独自価値
- PDF出力・共有リンク・進行モードの3つのアクションを常時表示

### 4. 進行モード（最大の差別化要素）

SessionLabにもButterにもない「対面ワークショップの当日プロンプター」:

- 全画面表示、大きな文字（会場でスマホ/タブレットを見ながら進行）
- カウントダウンタイマー付き
- スワイプで次のエクササイズへ遷移
- 「声かけ例」「注意点」がリアルタイム表示
- 時間超過/短縮時の調整提案
- **これだけで「計画ツール」→「実行ツール」に進化する**

### 5. エクササイズライブラリ画面（新設）

SessionLabのライブラリUIを参考に、独自の強みを加える:

- カード型グリッド表示
- フィルター: カテゴリ / レベル / 人数 / 所要時間
- 各カードに: 名前、所要時間、人数範囲、レベルバッジ、空間配置アイコン
- 検索機能
- お気に入り登録

## デザイントークン統一方針

### カラーパレット（削減・統一）

```
プライマリ:    #1A1A1A（テキスト）
セカンダリ:    #6B6B6B（補助テキスト）
背景:          #FAFAF8（暖かいオフホワイト）
カード背景:    #FFFFFF
アクセント:    #2A9D8F（ティール - ブランドカラー）
アクセント2:   #C8553D（テラコッタ - 強調用）
成功:          #4CAF50
警告:          #F59E0B
```

※目的別の6色は、ライブラリのタグ表示にのみ使用。メインUIには持ち込まない

### スペーシング（8pxグリッド）

```
4px   - アイコンとラベルの間
8px   - 要素内の最小パディング
16px  - カード内パディング
24px  - セクション間の小間隔
32px  - セクション間の標準間隔
48px  - セクション間の大間隔
64px  - ページセクション間
```

### タイポグラフィ

```
フォント: Noto Sans JP（メイン）、DM Mono（数値・タイムコードのみ）

見出しH1:  32px / 700weight / 1.3行間
見出しH2:  24px / 700weight / 1.4行間
見出しH3:  18px / 600weight / 1.4行間
本文:      16px / 400weight / 1.6行間
補足:      14px / 400weight / 1.5行間
キャプション: 12px / 500weight / 1.4行間
```

### タッチターゲット

```
ボタン最小サイズ:    48px × 48px
スライダーつまみ:    44px
スライダートラック:  12px高さ
タップ可能領域の間隔: 最低8px
```

## 競合に勝てる3つの武器（UI観点）

### 1.「30秒で完成」の体験
SessionLabはゼロから手動設計。我々は自動生成+編集。圧倒的に速い。
→ ファーストビューで「30秒」を強調し、体験させる

### 2. 進行モード
計画→実行が1つのアプリで完結。競合は計画止まり。
→ ファシリテーターの「現場の相棒」というポジションを取る

### 3. 空間配置図 + トランジションガイド
演劇のプロだからこそ提供できる、身体的アクティビティ特有の情報。
→ SessionLabのデジタル活動中心のライブラリとの明確な差別化

## 実装優先順位

### Phase 1: 即着手（現状の致命的課題を解消）
1. ヒーローセクション追加（ファーストビュー改善）
2. PDF/画像エクスポート機能
3. LocalStorageによるカリキュラム保存
4. 共有リンク生成（URLパラメータでカリキュラム復元）
5. デザイントークン統一（カラー・スペーシング・タッチターゲット）

### Phase 2: 差別化（1-2ヶ月）
6. 生成結果のドラッグ＆ドロップ編集
7. エクササイズ差し替えUI
8. 進行モード（タイマー付きプロンプター）
9. トランジションガイド表示
10. エクササイズライブラリ画面

### Phase 3: 成長（3-6ヶ月）
11. 感情曲線の可視化
12. 空間配置図
13. ユーザーアカウント + フリーミアム
14. テンプレート共有コミュニティ
15. 多言語対応
