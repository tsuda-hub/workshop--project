# Workshop Market Research - Theater Method Lab

## 現状のアプリ概要

- **名称**: Theater Method Lab
- **URL**: https://workshop-project-mocha.vercel.app/
- **コンセプト**: 演劇的手法を活用したワークショップ自動生成ツール
- **基本フロー**: パラメータ入力（人数・目的・レベル・時間）→ カリキュラム自動生成 → 詳細表示
- **技術構成**: React（単一JSXファイル）、Vercelデプロイ
- **エクササイズ数**: アイスブレイク5種、メインワーク25種（5カテゴリ）、クロージング4種

---

## 市場ポジション

### 直接競合: 存在しない（ブルーオーシャン）

演劇的手法・インプロ・身体表現を用いたワークショップの計画・実行に特化したデジタルツールは現時点で確認されていない。

### 間接競合

| ツール | 特徴 | 弱点（我々の機会） |
|--------|------|---------------------|
| **SessionLab** | ワークショップ設計特化、700+手法ライブラリ、タイムライン、共同編集 | デジタル活動中心。身体的・空間的アクティビティに未対応 |
| **Butter** | リアルタイムオンラインワークショップ実行 | 計画機能が弱い。対面向けではない |
| **Howspace** | AI活用、大規模対応、非同期ファシリテーション | 高価格。体験型ワークには不向き |
| **Miro / Mural** | ビジュアルコラボレーション、テンプレート豊富 | ワークショップ「設計」には特化していない |
| **TeamBuilding.com / Confetti** | チームビルディングアクティビティ予約 | カタログ型。カスタマイズ性なし |

### 日本市場の特徴

- 主要ツールはほぼ英語圏向け。日本語UIやテンプレートが不足
- 日本の研修文化（起承転結型構成、上下関係への配慮）に最適化されたツールがない
- 企業研修における体験型学習の採用が増加傾向

---

## 成功しているツールの共通要素

### UI/UX

- タイムラインビュー（時間軸に沿ったアクティビティ配置）
- ドラッグ＆ドロップ（直感的な並べ替え・時間調整）
- テンプレートライブラリ（ゼロから作らなくてよい安心感）
- リアルタイム共同編集
- PDF/Wordエクスポート

### 採用を促進する要素

- 無料プランの存在
- コミュニティ/マーケットプレイス（手法・テンプレート共有）
- 既存ツールとの連携（Zoom、Slack、Google Calendar）
- 5分以内で最初のワークショップ計画を作れるオンボーディング体験

---

## 市場のギャップ（我々が埋められるもの）

1. **「計画」と「実行」の断絶** - 計画ツールと実行ツールが別々。シームレスなサイクルがない
2. **身体的・体験型アクティビティへの対応不足** - 既存ツールはデジタル上の活動が前提
3. **参加者の感情曲線設計** - エネルギーレベル・感情の起伏を意識した設計支援がない
4. **トランジション（つなぎ）のガイダンス不足** - アクティビティ間の導入トークの支援がない
5. **ファシリテーターの暗黙知のデジタル化** - 「場の読み方」「臨機応変な対応」のサポートがない
6. **対面ワークショップの当日運営支援** - リアルタイム進行ガイド、タイマー、参加者連携がない

---

## トレンド

- **AI活用**: 目的に応じた自動提案、リアルタイム要約、パーソナライズ推薦
- **ハイブリッド対応**: 対面+リモート同時参加の設計
- **ゲーミフィケーション**: ポイント、バッジ、競争/協力要素
- **非同期ファシリテーション**: 事前ワーク→同期セッション→事後リフレクション
- **データドリブン**: エンゲージメント測定、効果のビジネス指標紐付け

## 必要な機能・UI改善ロードマップ

### Phase 1: MVP強化（すぐ着手）

| 機能 | 目的 |
|------|------|
| PDF/画像エクスポート | 印刷して現場に持っていける形。ファシリテーターは紙で確認する |
| カリキュラム保存（LocalStorage） | 生成結果が消えない。継続利用の基盤 |
| 共有リンク生成 | URLでチームに共有→口コミの起点 |
| アクティビティの手動入れ替えUI | 生成結果を自分好みにカスタマイズできる柔軟性 |

### Phase 2: 差別化（1-2ヶ月）

| 機能 | 目的 |
|------|------|
| 当日進行モード（プロンプター） | タイマー付き、スワイプで次へ。スマホで使える現場ツールに変化 |
| トランジションガイド | アクティビティ間の「声かけ例」「導入トーク」表示。初心者に必須 |
| 感情曲線の可視化 | ワークショップ全体の感情的起伏をグラフ表示。演劇ならではの差別化 |
| ユーザーアカウント＋フリーミアム | 無料:月3回生成 / 有料:無制限+PDF+保存 |
| エクササイズライブラリの拡充 | 現在34種→目標100種以上 |

### Phase 3: 成長エンジン（3-6ヶ月）

| 機能 | 目的 |
|------|------|
| AI最適化 | ランダム選択→インテリジェント選択（目的・参加者特性に基づく最適組み合わせ） |
| 自然言語カスタマイズ | 「もっと体を動かすものに」「恥ずかしがり屋が多い」等の入力で調整 |
| テンプレート共有コミュニティ | ファシリテーターが自作カリキュラムを投稿・共有 |
| 参加者向けQRコード/投票機能 | 参加者がスマホでリアルタイム参加・感想入力 |
| 効果測定アンケート | ワークショップ後のフィードバック収集・集計 |
| 多言語対応（英語） | グローバル市場へのリーチ |
| 空間レイアウトビュー | 各エクササイズの空間配置図（円形、ペア、全体等）を表示 |

---

## 差別化の核心

**「演劇×ワークショップ設計」という直接競合が存在しない領域を押さえること。**

SessionLabはデジタル活動中心。身体的・空間的アクティビティの設計には対応していない。この空白地帯が最大の武器。

### 我々だけが提供できる価値

1. 演劇的手法に特化したエクササイズライブラリ
2. 感情曲線を意識したカリキュラム設計
3. 身体的アクティビティの空間配置ガイド
4. 心理的安全性の段階的構築を支援する設計
5. 日本語ファーストのワークショップ計画ツール
